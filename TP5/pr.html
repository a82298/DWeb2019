    • ABSTRACT

    Foi proposto aos alunos que fosse feito um servidor que respondesse a um pedido GET ('/index' ou '/' ou '/gestaoTarefas'), em que o objetivo perante este pedido será apresentar uma página onde será possível acrescentar tarefas novas (através de um pedido PUT) e listar as tarefas que já foram registadas anteriormente. O servidor caso seja pedido algo que não é suportado, deverá responder com uma página de erro.

    Portanto, para realizar o trabalho, foram usados 3 ficheiros pug. O layout.pug que irá ser 'extendido' pelos outros dois ficheiros pug e foi mostrado pelo professor na última aula. O ficheiro erro.pug será o template para apresentar uma página de erro quando o pedido não é suportado pelo servidor. O ficheiro index.pug deverá ser responsável por apresentar a página principal.

    O ficheiro gestaoTarefas.js deverá processar os pedidos e responder-lhes de maneira adequada com o ficheiro correto conforme esses pedidos.

    DELIVERABLES

  • Ficheiro PUG que será extendido
  • Ficheiro PUG que será responsável por apresentar erros
  • Ficheiro PUG que será responsável por apresentar a página principal
  • Servidor que responde aos pedidos
  • Ficheiro XML transformado para realizar este relatório
